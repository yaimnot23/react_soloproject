# React Solo Project

# Project: Bug Hunter (나만의 에러 현상수배소)

> "똑같은 에러로 또 시간 낭비하지 말자!"
> 
> 
> 개발 중 발생하는 에러를 기록하고, 자주 발생하는 실수를 파악하며, 해결 과정을 블로그 포스팅 형식으로 쉽게 변환해 주는 생산성 도구입니다.
> 

## 1. 기술 스택 (Tech Stack)

빠른 프로토타입 완성을 위해 설정이 간편하고 생산성이 높은 스택을 선정했습니다.

- **Framework**: Next.js 14+ (App Router)
- **Language**: TypeScript
- **Database**: SQLite (파일 기반 DB, 별도 설치 불필요)
- **ORM**: Prisma (DB 제어 및 스키마 관리)
- **Styling**: Tailwind CSS (빠른 UI 구축)

## 2. 핵심 기능 명세 (Features)

### ✅ A. 에러 수집 및 관리 (Basic CRUD)

- [ ]  **현상수배 (Create)**: 에러 제목, 상세 내용, 발생 상황을 빠르게 기록합니다.
- [ ]  **검거 상태 관리 (Update)**:
    - `수배 중` (미해결, 빨간색 뱃지)
    - `검거 완료` (해결됨, 초록색 뱃지)
- [ ]  **증거 인멸 (Delete)**: 실수로 작성한 로그를 삭제합니다.

### ✅ B. 유사 에러 카운터 (Frequency Counter)

- [ ]  **"또 너냐?" 버튼**: 리스트에서 버튼 클릭 시 `발생 횟수 +1` 증가.
- [ ]  **최신화**: 카운트 증가 시 `최근 발생 시간`을 현재 시간으로 갱신.
- [ ]  **우범 순위 정렬**: 자주 발생하는 에러 순(`Count` 내림차순)으로 정렬하여 취약점 파악.

### ✅ C. 블로그 글감 내보내기 (Markdown Export)

- [ ]  **원클릭 복사**: 해결 완료된 에러 상세 페이지에 `[블로그용 복사]` 버튼 제공.
- [ ]  **포맷팅**: 버튼 클릭 시 아래 양식으로 클립보드에 자동 복사.
    

    ### ✅ D. 검색 및 분류 (Search & Filter) - *NEW!*

- [ ]  **통합 검색**: 에러 제목 또는 상세 내용 키워드로 과거 기록을 검색합니다.
- [ ]  **태그(Tag)**: `#React`, `#DB` 등 태그를 지정하여 관련 기술 스택별로 모아봅니다.


    ```
    # [Trouble Shooting] <에러 제목>
    
    ## 1. 문제 상황
    <에러 상세 내용>
    
    ## 2. 해결 방법
    <해결 방법(Solution)>
    
    > 💡 Bug Hunter에 의해 <N>번 발생했던 에러입니다.
    
    ```
    

## 3. 🗄 데이터베이스 설계 (DB Schema)

`prisma/schema.prisma` 파일에 적용할 모델입니다.

```
model BugLog {
  id              Int      @id @default(autoincrement())

  // 1. 기본 정보
  errorSubject    String   // 에러 제목 (예: NullPointerException 발생)
  errorDetail     String?  // 에러 상세 로그 (선택사항)
  solution        String?  // 해결 방법 (메모)

  // 2. 상태 관리
  isSolved        Boolean  @default(false) // 해결 여부 (false: 미해결, true: 해결)

  // 3. 카운터 기능 (핵심 기능 B)
  occurrenceCount Int      @default(1)     // 발생 횟수 (기본값 1)

  // 4. 시간 정보
  createdAt       DateTime @default(now()) // 최초 발생 시간
  lastOccurredAt  DateTime @default(now()) // 최근 발생 시간 (정렬용)
}

```

## 4. 📱 화면 구성 (UI Wireframe)

### 1️⃣ 메인 페이지 (`/`)

- **Header**: 프로젝트 타이틀 및 로고
- **Input Area**: "새로운 에러 등록" (제목만 입력하여 빠른 등록)
- **Filter Tab**:
    - `전체 보기`
    - `자주 발생하는 순` (occurrenceCount 내림차순)
    - `미해결만 보기`
- **List Area**: 에러 카드 리스트
    - 카드 우측에 `🚨 +1` 버튼 배치 (재발 시 클릭용)
    - 해결 여부에 따른 시각적 구분 (배경색 등)

### 2️⃣ 상세 페이지 (`/logs/[id]`)

- **Content Area**:
    - 에러 상세 내용 수정 (Textarea)
    - 해결 방법 기록 (Textarea)
- **Action Bar**:
    - `[해결 완료]` 체크박스
    - `[📋 마크다운으로 복사]` 버튼 (Alert: "복사되었습니다!")
    - `[삭제]` 버튼

## 6. 🔄 사용 시나리오 (User Scenario)

개발 중 에러가 발생했을 때 애플리케이션을 활용하는 구체적인 흐름입니다.

### 1단계: 현상수배 (등록)

- **상황**: 코딩 중 에러(예: Python IndentationError) 발생.
- **행동**: 해결책을 찾기 전, 메인 페이지 입력창에 에러 제목만 입력하여 빠르게 등록.
- **결과**: 목록에 '수배 중(미해결)' 상태로 추가됨. 발생 횟수 1회.

### 2단계: 수사 진행 (기록)

- **상황**: 검색을 통해 에러 원인 파악.
- **행동**: 상세 페이지로 이동하여 에러 로그(문제 상황)와 원인, 해결 방법을 상세히 기록.

### 3단계: 검거 완료 (해결)

- **상황**: 코드 수정 후 에러 해결.
- **행동**: '해결 완료' 체크박스 선택.
- **결과**: 상태가 '검거 완료(해결됨)'으로 변경됨.

### 4단계: 재범 발생 (재발 방지)

- **상황**: 며칠 뒤 동일한 에러 재발생.
- **행동**: 새로운 항목을 생성하지 않고, 기존 목록에서 해당 에러의 '+1' 버튼 클릭.
- **결과**: 발생 횟수가 증가하고 '최근 발생 시간'이 갱신됨. 자주 발생하는 에러 순으로 정렬 시 상단에 노출되어 취약점 파악 가능.

### 5단계: 지식 공유 (블로그 작성)

- **상황**: 해결 과정을 기술 블로그에 정리.
- **행동**: 상세 페이지의 '블로그용 복사' 버튼 클릭.
- **결과**: 문제 상황, 해결 방법, 발생 횟수가 포함된 마크다운 양식이 클립보드에 복사됨.
